在 macOS ARM 上设置 Hexo 博客框架并将生成的博客发布到 GitHub Pages

7月 29 2023 Hexo 6 分钟 读完 (约 868 字)

近日开始萌生写博客，把平时自己的活动记录下来。想到了以前用过的 Hexo，一个快速、简洁且高效的博客框架，并且支持一键部署到 GitHub Pages。使用 GitHub Pages 配合自定义域名，等于是有了一个完整的博客网站。

以下记录了我在 Mac Mini M1 设备上安装、配置 Hexo，在 Github 上新建 repo，把 Hexo 生成的网站部署到 repo，并发布到 GitHub Pages 上的过程。

1. 安装 Node.js

安装 Hexo 之前先确保安装了 Node.js。如果之前已安装 Node.js，可以跳到第二步。

在 macOS 上可以使用 Homebrew 安装 Node.js：

同时可能需要安装 npm，因为我发现 Homebrew 安装的 Node.js 并不自带 npm：

确认一下 Node.js 的版本，在终端输入 node --version，在我写这篇博客时，Homebrew 安装的最新版本是 v20.5.0。

2. 安装 Hexo

确认 Node.js 已经安装之后就可以使用 npm 安装 hexo 包了。

4. 配置站点

生成网站之后，首先配置 Hexo 的 _config.yml 文件。

打开 _config.yml 文件后需要设置的有站点的名称、url、语言等。对于语言设置，因为我想我的博客同时有英文和中文内容，所以可以设置为 language: ['en', 'zh-cn']，前面的 en 是默认的。 同时，找到 i18n_dir 国际语言设置设置，将其设置为 i18n_dir: :lang, 意为通过语言编码目录来识别语言。所以，如果是中文的 post 的话，记得把 post 的源文件放到 zh-cn 目录。

接下来需要安装及指定网站所用的主题。因为支持多语言的主题我只找到 minos 一个。所以我用的主题是 minos。theme: minos

6. 自定义域名

部署完成后，可以在 Github Pages 设置自定义域名。设置，完成后，在仓库的 gh-pages 分支的根目录会生成一个名为 CNAME 的文件，里面就是之前设置的自定义域名。但是这个文件在每次在 Hexo 部署时就会消失，原因是 Hexo 本地并没有这个文件。为避免每次部署都要重新设置自定义域名，可以在本地的 source 文件夹放置一个 CNAME 文件，这样每次部署时 CNAME 都会一同被上传。
